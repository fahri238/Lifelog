# **LifeLog Project – Complete Guide**

## **1. Project Overview**

LifeLog is a personal activity dashboard application designed to help users record, track, and review their daily activities such as studying, exercise, reading, or other routines.
This project is ideal for practicing JavaScript in a *project-based learning* approach, covering DOM manipulation, OOP, asynchronous programming, API integration, modular architecture, and modern tooling.

---

## **2. Learning Objectives**

By building this project, you will learn to:

* Manipulate the DOM efficiently
* Apply object-oriented programming in real applications
* Manage global state and data flow
* Work with asynchronous JavaScript and APIs
* Structure code using ES Modules
* Use helper utilities and clean architecture patterns
* (Optional) Prepare the project for modern bundlers like Parcel/Vite/Webpack

## **4. Folder Descriptions**

### **Root Directory**

* **index.html** — main HTML file
* **style.css** — optional global stylesheet
* **script.js** — minimal entry file (imports modules when not using a bundler)

---

### **assets/**

Contains all static, non-code files:

* `icons/` → SVG/PNG icons
* `images/` → photos, covers, illustrations
* `fonts/` → custom font files

---

### **src/**

The main working directory for development.

---

### **src/js/model/**

Logic related to data representation and state.

* **Activity.js**
  Class blueprint for a single activity.

* **ActivityManager.js**
  Handles activity CRUD operations, sorting, filtering, and business logic.

* **state.js**
  Global application state object:

  ```js
  export const state = {
    activities: [],
    weather: {},
    settings: {}
  };
  ```

---

### **src/js/views/**

UI components responsible for rendering and collecting user input.

* **ActivityView.js** — renders the activity list
* **FormView.js** — manages input form interactions
* **WeatherView.js** — displays weather information
* **DashboardView.js** — summary, analytics, statistics, optional charts

> Views only handle UI and DOM; they do not contain business logic.

---

### **src/js/controllers/**

* **controller.js**
  The core application logic.
  Connects **model ↔ view**, handles all events such as:

  * Adding an activity
  * Removing/editing activities
  * Fetching weather data
  * Initial rendering
  * Updating UI state

---

### **src/js/helpers/**

Reusable helper functions.

* **api.js** — fetching external APIs
* **format.js** — formatting dates, numbers, and durations
* **utils.js** — utilities (UUID generator, localStorage wrapper, cloning, etc.)

---

### **src/js/config.js**

Contains configuration constants.

Example:

```js
export const API_KEY = 'YOUR_API_KEY';
export const WEATHER_ENDPOINT = 'https://api.openweathermap.org/...';
export const TIMEOUT_SEC = 5;
```

---

### **src/css/**

For styling and UI structure.

* **base.css** — resets, variables, typography
* **layout.css** — grid and layout structure
* **components.css** — cards, buttons, form styles
* **themes.css** — light/dark mode

---

### **src/data/**

Optional sample/testing data.
Useful when API calls or inputs are not ready.

---

### **dist/**

Generated by a bundler (optional).
Contains minified production code.

## **5. Development Roadmap**

### ✅ **Phase 1 — Core Features**

* Activity form
* Save activity
* Render list
* Edit & delete
* Save to localStorage

---

### ✅ **Phase 2 — OOP Refactor**

* Build `Activity` class
* Build `ActivityManager` class
* Rewrite logic using structured classes

---

### ✅ **Phase 3 — Weather API Integration**

* Get user location (geolocation API)
* Fetch weather data (OpenWeather or any API)
* Display weather in UI

---

### ✅ **Phase 4 — Utilities & Time Handling**

* Calculate activity duration
* Format time/date
* Weekly/monthly summaries

---

### ✅ **Phase 5 — Modular Architecture**

* Split JS files into ES Modules
* Use imports/exports correctly
* Organize logic according to MVC-like structure

---

### ✅ **Phase 6 — Advanced UI Enhancements**

* Sorting (by date, duration, category)
* Filtering
* Search bar
* Animations (fade, slide, transitions)
* Dark/Light mode
